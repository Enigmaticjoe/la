# Portainer CE Installer for Fedora 43

A robust, production-ready installer script for Docker Engine and Portainer Community Edition on Fedora 43.

## Features

- ✅ **Fedora 43 Compatible**: Uses updated DNF5 syntax with automatic fallback
- ✅ **Comprehensive Error Handling**: Cleanup on failure, detailed error messages
- ✅ **Multiple Installation Methods**: Tries 4 different repository configuration methods
- ✅ **Complete Docker Installation**: Includes Docker Compose, Buildx plugins
- ✅ **Portainer CE**: Ready-to-use container management interface
- ✅ **Security Hardened**: Root checks, Fedora version validation
- ✅ **User-Friendly**: Color-coded output, progress indicators, helpful summary

## Requirements

- **OS**: Fedora 43 (or Fedora 39+)
- **Privileges**: Root/sudo access
- **Network**: Internet connection for package downloads
- **Ports**: 9000 (HTTP) and 9443 (HTTPS) available

## What Gets Installed

### Docker Components
- **Docker Engine** (docker-ce)
- **Docker CLI** (docker-ce-cli)
- **Containerd** (containerd.io)
- **Docker Buildx Plugin** (docker-buildx-plugin)
- **Docker Compose Plugin** (docker-compose-plugin)

### Portainer
- **Portainer CE** (Community Edition)
- Exposed on ports 9000 (HTTP) and 9443 (HTTPS)
- Persistent volume for configuration

## Fedora 43 DNF Compatibility

This script addresses the DNF5 syntax changes in Fedora 43:

| Method | Command | Status |
|--------|---------|--------|
| 1 | `dnf config-manager add-repo URL` | Primary (DNF5) |
| 2 | `dnf config-manager addrepo --from-repofile=URL` | Fallback 1 |
| 3 | `dnf config-manager --add-repo URL` | Fallback 2 (Legacy) |
| 4 | `curl -fsSL URL -o /etc/yum.repos.d/docker-ce.repo` | Manual fallback |

The script automatically tries each method in order until one succeeds.

## Usage

### Running the Installer

```bash
sudo ./2.sh
```

The script will:
1. ✅ Check for root privileges
2. ✅ Verify Fedora version
3. ✅ Remove old Docker versions
4. ✅ Install dependencies (dnf-plugins-core)
5. ✅ Add Docker repository (with DNF5 compatibility)
6. ✅ Install Docker Engine and plugins
7. ✅ Start and enable Docker service
8. ✅ Verify Docker installation
9. ✅ Install Portainer CE
10. ✅ Configure firewall (if firewalld is active)
11. ✅ Add user to docker group

### After Installation

1. **Access Portainer**:
   - HTTP: `http://localhost:9000`
   - HTTPS: `https://localhost:9443`
   - Or use your server's IP address

2. **Create Admin Account**:
   - Open Portainer in your browser
   - Create your admin credentials on first visit

3. **Apply Group Changes** (if not root):
   ```bash
   # Log out and back in, or run:
   newgrp docker
   ```

## Troubleshooting

### Script Fails at Repository Configuration

**Error**: `Unknown argument "--add-repo" for command "config-manager"`

**Solution**: This script handles this automatically by trying multiple methods. If all fail:
```bash
# Check DNF version
dnf --version

# Manually add repository
sudo curl -fsSL https://download.docker.com/linux/fedora/docker-ce.repo \
  -o /etc/yum.repos.d/docker-ce.repo

# Retry installation
sudo ./2.sh
```

### Docker Service Won't Start

```bash
# Check Docker status
sudo systemctl status docker

# View Docker logs
sudo journalctl -u docker.service

# Restart Docker
sudo systemctl restart docker
```

### Portainer Container Not Running

```bash
# Check container status
docker ps -a | grep portainer

# View Portainer logs
docker logs portainer

# Restart Portainer
docker restart portainer
```

### Permission Denied (Docker)

```bash
# Add user to docker group
sudo usermod -aG docker $USER

# Apply group changes
newgrp docker

# Or log out and back in
```

### Firewall Blocking Portainer

```bash
# If firewalld is running
sudo firewall-cmd --permanent --add-port=9000/tcp
sudo firewall-cmd --permanent --add-port=9443/tcp
sudo firewall-cmd --reload

# Check firewall status
sudo firewall-cmd --list-ports
```

## Security Considerations

### Root Privileges
The script requires root/sudo to:
- Install system packages
- Configure system services
- Modify firewall rules
- Add users to groups

### Firewall Configuration
The script automatically configures firewalld if it's running. For production:
```bash
# Restrict to specific IPs
sudo firewall-cmd --permanent --add-rich-rule='rule family="ipv4" \
  source address="192.168.1.0/24" port protocol="tcp" port="9000" accept'
```

### Docker Socket Access
Portainer requires access to `/var/run/docker.sock` for container management. This is expected and necessary for Portainer to function.

### Network Exposure
By default, Portainer is exposed on all interfaces (0.0.0.0). For production:
- Use a reverse proxy (nginx, Traefik) with SSL
- Restrict access with firewall rules
- Consider using Portainer's built-in HTTPS on port 9443

## Uninstallation

### Remove Portainer Only

```bash
# Stop and remove Portainer container
docker stop portainer
docker rm portainer

# Remove Portainer volume (optional)
docker volume rm portainer_data
```

### Remove Docker Completely

```bash
# Stop Docker service
sudo systemctl stop docker

# Remove Docker packages
sudo dnf remove -y docker-ce docker-ce-cli containerd.io \
  docker-buildx-plugin docker-compose-plugin

# Remove Docker data (optional - this deletes all containers, images, volumes)
sudo rm -rf /var/lib/docker
sudo rm -rf /var/lib/containerd

# Remove repository configuration
sudo rm /etc/yum.repos.d/docker-ce.repo
```
